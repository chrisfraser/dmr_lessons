DMR Radio Basics Reference · Glossary

DMR Glossary

The canonical vocabulary for this course. Every lesson uses these exact terms. Definitions tighten as understanding deepens.

This glossary covers Digital Mobile Radio (DMR) as it relates to bridging DMR voice onto a PoC (Push-to-talk over Cellular) platform. Terms are added here only once they are understood — it is a record of compressed knowledge, not a dictionary.

Core structure

DMR — Digital Mobile Radio
An open digital two-way radio standard defined by ETSI (TS 102 361), built on 2-slot TDMA in a 12.5 kHz channel.
Avoid: "digital walkie-talkie", "MOTOTRBO" (that's one vendor's brand of DMR).
TDMA — Time Division Multiple Access
Splitting one radio frequency into repeating time slots so multiple conversations share it. DMR uses two slots.
Avoid: "multiplexing" (too vague).
Time slot — TS1 / TS2
One of the two 30 ms turns into which a DMR frequency is divided. A single frequency carries two independent conversations, one per slot.
Avoid: "channel" (a slot is part of a channel, not the whole).
Color code — CC, 0–15
A number (0–15) that must match between radio and repeater for access — the digital equivalent of an analog PL/CTCSS tone. Keeps nearby systems on the same frequency from interfering.
Avoid: "channel ID", "encryption" (it is neither).
Talkgroup — TG, "Group"
A numeric ID identifying a virtual group of users. Your radio only hears traffic for the talkgroups it subscribes to. The unit we map onto a PoC group.
Avoid: "channel", "frequency".
Channel
A saved bundle of settings the operator selects as one unit: RX frequency, TX frequency, time slot, color code, and talkgroup. Selecting a channel sets all of these at once.
Avoid: "frequency" (a channel is more than a frequency).
Call type — Group / Private / All Call
Who a transmission addresses: a Group Call (everyone on a talkgroup), a Private Call (one specific Radio ID), or an All Call (one-way to everyone on the system).
All Call — ID 16777215 (0xFFFFFF)
A one-way transmission to every radio on the system, addressed to the reserved all-ones destination ID. Originated by keying a channel/contact whose TX contact is that ID; restricted per-radio by what's programmed in the code plug, not by the network. Radios generally always receive it regardless of their group list.
Avoid: "broadcast to a talkgroup" (it bypasses talkgroups entirely).

Infrastructure

Repeater
A fixed station that receives on one frequency and re-transmits on another to extend range. A DMR repeater synchronises both time slots, carrying two conversations at once.
Hotspot
A low-power personal device (often Pi-Star/MMDVM) that connects a single radio to a DMR network over the internet — a miniature repeater for the desk.
DMR network — Brandmeister, DMR-MARC, DMR+
Internet-connected infrastructure that routes talkgroups between repeaters and hotspots worldwide via master servers.
Radio ID — DMR ID
A unique number identifying a specific radio/operator, issued centrally by RadioID.net and used across all major networks. Addresses Private Calls. The field is 24-bit: values 0–16,777,215. Reserved: 0 (null) and 16777215 (All Call); usable individual IDs are ~16.7 million. No numeric overflow is possible — the limit is address-space exhaustion, not wraparound.
Avoid: "callsign" (the ID is numeric and separate from any callsign).
Code plug
The complete configuration file loaded into a radio — every channel, zone, scan list, contact and setting. Contains zones, which contain channels.
Avoid: "firmware" (the code plug is config, not the operating software).
Zone
A named bucket of channels in the code plug (usually by location or purpose) that re-populates the channel selector when chosen. Pure organisation, not over the air.
Avoid: "channel group", "talkgroup" (a zone groups channels, not users).
Scan list
An operator-defined list of channels the radio steps through, stopping on whichever has activity — how a user monitors several talkgroups at once.
Tier — I / II / III
DMR's three profiles: Tier I licence-free simplex; Tier II licensed conventional 2-slot repeaters (the common case for us); Tier III trunked multi-site systems.

Air interface (radio ↔ repeater)

Burst
One slot's worth of transmitted bits: 264 bits in 27.5 ms, laid out as 108-bit payload + 48-bit centre + 108-bit payload. The centre holds either a sync pattern or embedded signalling.
Voice superframe — bursts A–F
Six consecutive voice bursts (A–F), 360 ms. Burst A carries the voice sync; B–F carry embedded Link Control. Speech is three 20 ms AMBE+2 frames per burst.
Voice LC header — Link Control
The PDU sent before voice (and repeated in bursts B–F) announcing source Radio ID, destination talkgroup/ID, and call type (FLCO). The metadata a bridge maps to PoC.
CACH — Common Announcement Channel
Signalling on the repeater's outbound stream, interleaved between slots, telling radios whether each slot is busy/idle and which slot is which — without decoding voice.
Admit criterion — Always / Channel Free / Color Code Free
The politeness rule a radio checks before keying. "Color Code Free" (key only if the matching repeater's slot is idle) is the recommended repeater setting. The closest thing to access control — a filter, not a handshake.
Hang time
The brief window after a transmission in which the repeater keeps the slot + talkgroup reserved so the same conversation can continue. Per-slot.
Late entry
Recovering a call's identity after tuning in mid-transmission, because the Link Control is repeated in every superframe (≤360 ms to recover).

Voice & bridging

Vocoder / AMBE+2
The proprietary codec (DVSI AMBE+2) that compresses DMR voice to ~3,600 bps. Decoding/re-encoding it to a cellular codec is transcoding.
Transcoding
Converting voice from one codec to another (e.g. AMBE+2 → cellular AMR/Opus). Each conversion loses quality — measured at >12% intelligibility loss in one DVSI study — so it is the central fidelity risk for our bridge.
Gateway / bridge
Infrastructure that connects DMR to another system (analog, another DMR network, or — for us — a PoC platform), routing talkgroups and converting voice as needed.
PoC — Push-to-talk over Cellular
Walkie-talkie-style voice carried over cellular/IP data rather than radio RF. The platform our product bridges DMR into.
